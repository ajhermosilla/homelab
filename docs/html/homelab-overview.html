Every service runs as a Docker container across three physical locations: a local Proxmox hypervisor, a NAS for storage and Git, and a VPS for the Tailscale control plane and external DNS. OPNsense provides firewall, DHCP, and VLAN segmentation. All nodes communicate over a Headscale-managed WireGuard mesh, with no ports exposed to the public internet except the Headscale control server.

Network Architecture

All DNS resolution stays private. Home devices query Pi-hole, which forwards to Unbound on OPNsense for recursive resolution against root servers. The VPS uses a separate AdGuard + Unbound stack. Tailscale MagicDNS serves *.cronova.dev records to all mesh nodes via Headscale extra_records.

Security Hardening

  • All containers: cap_drop: [ALL] with explicit cap_add for required capabilities
  • All containers: security_opt: [no-new-privileges:true]
  • Stateless containers run with read_only: true rootfs
  • Authelia SSO with TOTP 2FA protecting 8 services
  • UFW with INPUT DROP policy on Docker VM
  • VPS exposes only the Headscale control port publicly
  • OPNsense gateway with VLAN segmentation (IOT, GUEST)
  • Private DNS: recursive resolution via Unbound (no third-party resolvers)
  • HSTS headers (max-age=31536000) on all Caddy-proxied services
  • All 23 Docker Compose stacks have .env.example files (secrets gitignored)
