Before Deployment Day

Remote prep from your MacBook

Step 1: Generate Secrets with the Prep Script

Run this from your homelab repo on the MacBook:

bash scripts/nas-prep-env.sh
What this does: Generates a random Samba password, creates .env files for the NAS Docker stacks, and prompts you for a Restic backup password.

Step 2: Generate a Tailscale Auth Key

ssh vps
docker exec headscale headscale preauthkeys create --expiration 1h
What this does: Creates a one-time key that lets the NAS join your Tailscale network. Expires in 1 hour, single use.
Timing tip This key expires in 1 hour. If you're doing prep the night before, skip this and do it right before Phase 4 on deployment day.

Step 3: Add DHCP Reservation in OPNsense

  1. Open SSH tunnel: ssh -L 8443:192.168.0.1:443 augusto@100.78.12.241
  2. Open https://localhost:8443
  3. Go to Services → DHCPv4 → LAN
  4. Add reservation: NAS MAC → 192.168.0.12
What this does: Tells OPNsense to always assign 192.168.0.12 to the NAS. Get the MAC from the motherboard sticker.

Step 4: Add DNS Entry in Pi-hole

ssh docker-vm
docker exec -it pihole bash

Inside the container, edit /etc/pihole/pihole.toml and add to the dns.hosts array:

[[dns.hosts]]
  addr = "192.168.0.12"
  names = ["nas.home", "syncthing.home"]

Then reload:

pihole reloaddns

Step 5: Flash Debian USB (if not done)

sudo dd if=debian.iso of=/dev/sdX bs=4M status=progress

Install Debian

~25 minutes

Boot from USB

  1. Connect USB keyboard, monitor, and Debian USB to the NAS
  2. Power on
  3. Press F8 (or DEL) for boot menu
  4. Select the USB drive

SoftwareSSH server + standard utilities only
What this does: Installs a minimal Debian server on the SSD. The two data drives are left untouched — we set those up in Phase 2.

First Boot Setup

Log in as root at the console:

apt update && apt upgrade -y
What this does: Downloads and installs the latest security patches.
apt install -y sudo curl wget git vim htop tmux
What this does: Installs essential tools for the rest of setup.
usermod -aG sudo augusto
What this does: Gives your user account permission to run admin commands with sudo.
reboot
Trouble? NAS didn't get IP 192.168.0.12

Check the OPNsense DHCP reservation. As a fallback, set a static IP as root:

nano /etc/network/interfaces

# Add:
auto enp0s25
iface enp0s25 inet static
  address 192.168.0.12
  netmask 255.255.255.0
  gateway 192.168.0.1
  dns-nameservers 192.168.0.10

Then reboot.

Set Up the Drives

~15 minutes

From here on, work via SSH: ssh augusto@192.168.0.12

Identify Your Drives

lsblk -f
What this does: Lists all drives. Expect: sda = Lexar SSD (boot), sdb = WD Purple 2TB, sdc = WD Red 8TB.

Check Drive Health

sudo apt install -y smartmontools
sudo smartctl -a /dev/sdb
sudo smartctl -a /dev/sdc
What this does: Runs a health check. Look for PASSED in the output.

Partition and Format

# Purple 2TB
sudo fdisk /dev/sdb
# n (new), p (primary), 1, Enter, Enter, w (write)
sudo mkfs.ext4 -L purple /dev/sdb1
# Red 8TB
sudo fdisk /dev/sdc
# n (new), p (primary), 1, Enter, Enter, w (write)
sudo mkfs.ext4 -L red8 /dev/sdc1

Create Mount Points

sudo mkdir -p /mnt/{purple,red8}

Configure Automatic Mounting

Get UUIDs:

sudo blkid

Edit fstab:

sudo nano /etc/fstab

# Add at the end (replace UUIDs with actual values):
UUID=<purple-uuid>  /mnt/purple  ext4  defaults,noatime  0  2
UUID=<red8-uuid>    /mnt/red8    ext4  defaults,noatime  0  2
What this does: Auto-mounts the drives on every boot. noatime improves performance.

Mount and Verify

sudo mount -a
df -h
What this does: Mounts everything now. You should see ~2TB on /mnt/purple and ~8TB on /mnt/red8.

Create Directory Structure

# Folders on Red 8TB
sudo mkdir -p /mnt/red8/{media,downloads,data,sync,backup}

# Folder on Purple 2TB
sudo mkdir -p /mnt/purple/frigate

# Shortcuts in /srv
sudo mkdir -p /srv
sudo ln -s /mnt/red8/media /srv/media
sudo ln -s /mnt/red8/downloads /srv/downloads
sudo ln -s /mnt/red8/data /srv/data
sudo ln -s /mnt/red8/sync /srv/sync
sudo ln -s /mnt/red8/backup /srv/backup
sudo ln -s /mnt/purple/frigate /srv/frigate

# Set ownership
sudo chown -R 1000:1000 /mnt/red8 /mnt/purple /srv
What this does: Creates the folder structure and convenient /srv shortcuts so services use consistent paths regardless of which physical drive holds the data.
Trouble? Drive not showing up 
lsblk
sudo fdisk -l

Check SATA cables. Try a different SATA port. Check health: sudo smartctl -H /dev/sdb

Install Docker

~10 minutes
curl -fsSL https://get.docker.com | sudo sh
What this does: Installs Docker using the official script.
sudo usermod -aG docker augusto
What this does: Lets your user run Docker without sudo.

Log out and back in:

exit
# SSH back in: ssh augusto@192.168.0.12
docker --version
docker compose version
docker run hello-world
What this does: Runs a tiny test container to confirm Docker works end-to-end.
Trouble? Docker won't install

Check internet: ping 8.8.8.8

Check DNS: ping google.com

If DNS fails, verify /etc/resolv.conf points to 192.168.0.10 (Pi-hole).

4

Connect to Tailscale

~5 minutes
Reminder If you generated the auth key earlier, it may have expired (1 hour). Generate a fresh one now from your MacBook: ssh vps then docker exec headscale headscale preauthkeys create --expiration 1h
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --login-server=https://hs.cronova.dev --authkey=<key>
What this does: Connects the NAS to your Tailscale network via Headscale. Replace <key> with your auth key.
tailscale status
tailscale ip
Note The NAS Tailscale IP is 100.82.77.97 (assigned by Headscale).
Trouble? Tailscale won't connect 
tailscale status

Auth key expired? Generate a new one. Re-authenticate:

sudo tailscale up --login-server=https://hs.cronova.dev --authkey=<new-key> --reset
5

Set Up NFS

~10 minutes
What NFS does: Lets the Docker VM mount the NAS's Frigate folder directly, as if it were a local drive. This is how Frigate stores camera recordings on the NAS.

Option A: Ansible (Recommended)

Run from your MacBook:

cd ~/homelab/ansible
ansible -i inventory.yml nas -m ping
ansible-playbook -i inventory.yml playbooks/nfs-server.yml -l nas
What this does: Automatically installs NFS, configures shared folders, opens firewall ports, and starts the service.
Option B: Manual setup (if Ansible isn't working) 
sudo apt install -y nfs-kernel-server
sudo nano /etc/exports

# Add:
/srv/frigate    192.168.0.10(rw,sync,no_subtree_check,no_root_squash)
/srv/media      192.168.0.0/24(ro,sync,no_subtree_check)
/srv/downloads  192.168.0.0/24(rw,sync,no_subtree_check)
/srv/backup     192.168.0.0/24(rw,sync,no_subtree_check)
sudo exportfs -ra
sudo exportfs -v
sudo systemctl enable nfs-kernel-server
sudo systemctl start nfs-kernel-server
sudo ufw allow from 192.168.0.0/24 to any port 111
sudo ufw allow from 192.168.0.0/24 to any port 2049
Trouble? Ansible can't reach NAS

Make sure ansible_host in inventory.yml matches the NAS's actual Tailscale IP from Phase 4.

Start the Services

~15 minutes

Storage Stack (Samba + Syncthing)

cd /opt/homelab/repo/docker/fixed/nas/storage

Copy the .env from your MacBook:

# On MacBook:
scp ~/homelab/docker/fixed/nas/storage/.env augusto@192.168.0.12:/opt/homelab/repo/docker/fixed/nas/storage/.env

Or create manually on the NAS:

cp .env.example .env
nano .env

# Set:
# TZ=America/Asuncion
# PUID=1000
# PGID=1000
# SAMBA_USER=augusto
# SAMBA_PASSWORD=<from-vaultwarden>
# MEDIA_PATH=/srv/media
# DATA_PATH=/srv/data
# DOWNLOADS_PATH=/srv/downloads
# BACKUP_PATH=/srv/backup
# SYNC_PATH=/srv/sync
docker compose up -d
docker compose ps
docker compose logs -f
What this does: Starts Samba (file sharing) and Syncthing (file sync) in containers. Press Ctrl+C to stop watching logs.

Backup Stack (Restic REST)

cd /opt/homelab/repo/docker/fixed/nas/backup

sudo mkdir -p /srv/backup/restic
sudo chown 1000:1000 /srv/backup/restic

Copy files from MacBook:

# On MacBook:
scp ~/homelab/docker/fixed/nas/backup/.env augusto@192.168.0.12:/opt/homelab/repo/docker/fixed/nas/backup/.env
scp ~/homelab/docker/fixed/nas/backup/htpasswd augusto@192.168.0.12:/opt/homelab/repo/docker/fixed/nas/backup/htpasswd

Or create manually:

cp .env.example .env
nano .env
# Set: BACKUP_DATA=/srv/backup/restic

sudo apt install -y apache2-utils
htpasswd -B -c htpasswd augusto
docker compose up -d
docker compose ps
curl http://localhost:8000/
What this does: Starts the Restic REST server for HTTP-based backups.
Trouble? Samba won't start 
docker logs samba

Check .env values. Test: smbclient -L localhost -U augusto

Check ports: ss -tlnp | grep -E '139|445'

Trouble? Restic REST won't start 
docker logs restic-rest

Check htpasswd file exists. Check .env has BACKUP_DATA=/srv/backup/restic.

Test: curl -v http://localhost:8000/

If Something Goes Wrong

Debian won't install (Phase 1)
  • Make sure you're booting from USB (F8 boot menu)
  • Try a different USB port
  • Re-flash the USB: sudo dd if=debian.iso of=/dev/sdX bs=4M status=progress
Drive won't mount after reboot (Phase 2)

Check fstab syntax: cat /etc/fstab

Verify UUIDs match: sudo blkid

Try manual mount: sudo mount /dev/sdb1 /mnt/purple

NFS not working (Phase 5) 
sudo systemctl status nfs-kernel-server
sudo exportfs -v
sudo ufw status

From Docker VM: showmount -e 192.168.0.12

Make sure firewall allows ports 111 and 2049.

Can't SSH into NAS
  • Connect keyboard/monitor directly
  • Check network: ip addr
  • Check SSH: systemctl status ssh
  • Try local network first, then Tailscale
Nuclear option: Full reinstall

If everything is broken:

  1. Boot from Debian USB
  2. Reinstall on the SSD only
  3. Your data drives (Purple and Red) are not touched — data is safe
  4. After reinstall, re-mount drives using their UUIDs (Phase 2)
