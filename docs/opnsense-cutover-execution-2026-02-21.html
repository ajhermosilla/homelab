OPNsense Gateway Cutover

Date: 2026-02-21
Window: 30 minutes max
Rollback: ~2 minutes
Alt internet: Phone tethering (safety net)

Before You Start

Step 1 — Connect nic1 to Switch

Plug Ethernet cable from Proxmox nic1 (2nd port) into MokerLink switch.

Verify:

ssh augusto@192.168.0.237 "ip link show nic1 | grep 'state UP'"

state UP

Step 2 — Apply Proxmox Network Config

Moves management IP from vmbr0 → vmbr1 (now on the switch via nic1).

ssh augusto@192.168.0.237 "sudo cp /etc/network/interfaces.cutover /etc/network/interfaces && sudo cp /etc/network/interfaces.d/vmbr1.cutover /etc/network/interfaces.d/vmbr1 && sudo ifreload -a"

Verify:

ssh augusto@192.168.0.237 "ip -br addr show vmbr0 && ip -br addr show vmbr1"

vmbr0 — no IP  |  vmbr1 — 192.168.0.237/24

Step 3 — Swap Cables

1. Unplug ISP Ethernet from TP-Link WAN → plug into Proxmox nic0
2. Unplug old switch cable from Proxmox nic0 (no longer needed)
Internet is now DOWN for everyone. Clock is ticking.

Step 4 — Verify OPNsense Gets Public IP

Open Proxmox web UI → VM 100 (OPNsense) → Console.

Dashboard → Interfaces → WAN → should show a public IP.

If no public IP after 30 seconds:

/usr/local/sbin/configctl interface reconfigure wan
If no public IP after 2 minutes → ROLLBACK (see bottom).

Step 5 — Switch TP-Link to AP Mode

From Mac (still connected to WiFi):

#Action
1Open http://192.168.0.1 (TP-Link admin)
2Operation Mode → Access Point
3Set static IP: 192.168.0.2
4Apply (TP-Link reboots, ~1 min)
5Move TP-Link cable: LAN port → switch (NOT WAN)
WiFi will briefly drop. Same SSID/password = auto-reconnect.

Step 6 — Configure OPNsense LAN

From Proxmox web UI → VM 100 → Console → browse to https://192.168.1.1:

6a. Change LAN IP

Interfaces → LAN → IPv4192.168.0.1/24
Save → Apply

6b. Enable DHCP

Services → DHCPv4 → LANEnable: ✓
Range192.168.0.100 – 192.168.0.250
DNS server192.168.0.10
Gateway192.168.0.1

6c. Static Mappings

DeviceMACIP
Docker VMBC:24:11:A8:E9:C5192.168.0.10
TP-Link AP(from lease list)192.168.0.2

Save → Apply.

After applying, OPNsense LAN is at 192.168.0.1. Proxmox gateway now works. Tailscale reconnects.

Step 7 — Re-IP Docker VM

Docker VM's gateway (192.168.1.1) is gone. Fix from Proxmox:

ssh augusto@192.168.0.237 "sudo qm guest exec 101 -- bash -c 'ip addr del 192.168.1.10/24 dev ens18; ip addr add 192.168.0.10/24 dev ens18; ip route add default via 192.168.0.1'"
If qm guest exec fails → use Proxmox web UI → VM 101 → Console, then run the 3 commands manually.

Make it persistent:

ssh augusto@192.168.0.237 "sudo qm guest exec 101 -- bash -c \"sed -i 's/192.168.1.10/192.168.0.10/g; s/gateway 192.168.1.1/gateway 192.168.0.1/g' /etc/network/interfaces\""

Verify:

ssh augusto@192.168.0.10 "ip addr show ens18 | grep inet && ping -c1 8.8.8.8"

Rollback — If Anything Goes Wrong

Before Step 6 (OPNsense LAN still 192.168.1.x)

# 1. Revert Proxmox config
ssh augusto@192.168.0.237 "sudo cp /etc/network/interfaces.original /etc/network/interfaces && sudo cp /etc/network/interfaces.d/vmbr1.original /etc/network/interfaces.d/vmbr1 && sudo ifreload -a"

# 2. Physical: ISP cable → TP-Link WAN, switch cable → Proxmox nic0, unplug nic1
# 3. TP-Link: reboot to restore router mode

After Step 6 (OPNsense LAN already at 192.168.0.x)

#Action
1OPNsense console: LAN → change back to 192.168.1.1/24, Apply
2Docker VM: revert IP to 192.168.1.10, gateway 192.168.1.1
3Then do the cable/config revert above
